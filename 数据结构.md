# 数据结构  
## 1. 绪论
### 1.1 什么是数据结构  
**数据结构**：描述非数据计算的操作对象以及他们的关系的学科  

### 1.2 基本概念和术语
**数据**： 能输入计算机且能被计算机处理的各种符号的集合

**数据元素**：数据的基本单位，在计算机程序中通常作为一个整体进行考虑和处理，也称元素、记录、结点或顶点  

**数据项**：构成数据元素的不可分割的最小单位  

**数据对象**：性质相同的数据元素的集合，是数据的子集  

**数据结构**：数据元素相互的关系  
1. 逻辑结构（数据之间的逻辑结构）
2. 存储结构（数据元素极其关系在内存中的实现）
3. 数据的运算和实现  

**逻辑结构**：  
1. 线性结构（有且只有一个开始和结束，并且每个节点最多只有一个直接前趋和一个直接后继）（一对一）
2. 非线性结构（一对多或多对一或多对多）

**存储结构**：
1. 顺序存储结构（一片连续的存储单元依次存储，逻辑关系由存储位置表示）
2. 链式存储结构（用一组任意的存储单元存储数据元素，数据元素之间的逻辑结构用指针来表示）  
3. 索引存储结构（在存储结点信息的同时还建立索引表）
4. 散列存储结构（根据结点的关键字直接计算出存储地址）  

**数据类型**：  
性质相同的值的集合以及定义在值集上的一组操作的总称  

**抽象数据类型**：  
提取数学特征，忽略内部实现  
定义：数据对象的定义，数据关系的定义，基本操作的定义

### 1.3 算法和算法分析
#### 1.3.1 算法
算法：对特定问题求解步骤的描述  
**算法的描述**：  
1. 自然语言
2. 流程图
3. 伪代码

程序等于数据结构加算法  

算法特性：  
1. 有穷性
2. 确定性
3. 可行性
4. 输入：零个或多个输入
5. 输出：一个或多个输出  
#### 1.3.2 算法设计的要求
算法设计的要求：  
1. 正确性（1. 不含语法错误 2. 程序对于几组输入数据能满足要求结果 3. 程序对于精心挑选的、典型的、苛刻且带有刁难性的输入能获得满足结果）
2. 可读性（人类易于理解）
3. 健壮性（输入非法数据时，算法恰当作出反应）
4. 高效性（尽量少的时间和存储）

#### 1.3.3 算法效率的度量
时间效率：  
算法的运行时间大致等于计算机执行一种简单操作所需的时间与算法中进行的简单操作次数乘积  
简化成算法关键部分的执行次数，即语句的频度

**时间复杂度**:  
比较频度的数量级，时间趋于无穷时与n的关系，找基本语句（执行次数最多）  

如果f(n)是m次多项式，则取最高次项    

时间复杂度由嵌套最深层语句的频度决定   

加法规则：取最大值  
乘法规则：取乘积  

空间效率：  
算法所需存储空间的度量 

## 2. 线性表
### 2.1 线性表的类型定义  
**线性表**：具有相同特性的数据元素的一个有限序列  
基本要素：线性起点、直接前趋、直接后继、线性终点  
**表长**：数据元素个数  

**空表**：n = 0  

顺序存储结构问题：
1. 存储空间分配不灵活
2. 空间复杂度高

### 2.2 线性表的顺序表示和实现(顺序表)  
逻辑相邻，物理也相邻  
线性表的第一个地址叫：**基地址**，**或首地址**  

**线性表顺序存储结构占用一片连续的存储空间**  

顺序表优点：任一元素可以随机存取  

线性表的顺序实现：  
```C
#define MAX_SIZE 100
typedef int ElemType;
typedef struct{
    ElemType elem[MAX_SIZE]; //存储元素的数组
    int length; //存储已经存储了几个元素
}SqList;
```

顺序表的初始化
```C
int InitList_Sq(SqList &L){
    L.elem = (ElemType*)malloc(sizeof(ElemType) * MAX_SIZE);
    L.length = 0;
    return 1;
}
```
顺序表的插入:  
自然语言描述：从后往前，把每个数都往后移一位，最后把要插入的数插进去，Length++  

顺序表的删除：  
自然语言描述：从前往后，把每个数往前移一位，Length--  

顺序表的查找：  
从前往后遍历，依次比较，返回位置  

### 2.3 线性表的链式表示和实现(链表)
**一个结点包含数据和下一个节点的地址**

单链表的结点的典型实现：  
```C
typedef int ElemType;
typedef struct node{
    ElemType data;
    struct node* next;
}Node;
```  

**单链表的插入**：  
把s插到p的后面为例  
s->next = p->next;   
p->next = s; 

**单链表的删除**：   
以删除p后面的结点为例  
s = p -> next;  
p->next = p -> next -> next; 

**单链表的插入**  
1. 头插法：每一次数据都插在头结点的后面  
典型实现：  
```C
int insertHead(Node* L, ELemType e){
    Node* p = (Node*)malloc(sizeof(Node));
    p -> data = e;
    p -> next = L -> next;
    L -> next = p;
}
```
2. 尾插法：每一次数据都插在尾结点的后面  
典型实现：  
```C
Node* get_tail(Node *L){
    Node* p = L;
    while(p -> next != NULL){
        p = p -> next;
    }
    return p;
}

Node* insertTail(Node *tail, ElemType e){
    Node* p = (Node*)malloc(sizeof(Node));
    p -> data = e;
    tail -> next = p;
    p -> next = NULL;
    return p;
}
```  
3. 在指定位置插入数据  
典型实现：  
```C
int insertNode(Node*L, int pos, ElemType e){
    Node*p = L;
    int i = 0;
    //找到插入位置的前驱结点
    while(i < pos - 1){
        p = p -> next;
        i++;
        if(p == NULL){
            return 0;
        }
    }

    //要插入的新节点
    Node *q = (Node*)malloc(sizeof(Node));
    q -> data = e;
    q -> next = p -> next;
    p -> next = q;
    return 1;
}
``` 
**单链表结点的删除**:  
典型实现:  
```C
int deleteNode(Node*L, int pos){
    Node *p = L;
    int i = 0;
    //找到要删除结点的前驱结点
    while(i < pos - 1){
        p = p -> next;
        i++;
        if(p == NULL){
            return 0;
        }
    }

    //鲁棒性
    if(p -> next == NULL){
        printf("要删除的位置错误\n");
        return 0;
    }

    //q指向要删除的结点
    Node *q = p -> next;
    p -> next = q -> next;
    free(q);
    return 1;
}
```

**释放链表**  
自然语言描述:  
1. 指针p指向头节点后的第一个结点  
2. 判断p是否指向空结点  
3. 如果p不为空,用指针q记录指针p的后继结点
4. 释放p指向的结点
5. 把q赋值给p
6. 循环操作  

典型实现:  
```C
void freeList(Node* L){
    Node * p = L -> next;
    Node * q;

    while(p != NULL){
        q = p -> next;
        free(p);
        p = q;
    }
    L -> next = NULL;
}
```
小总结: 读取数据顺序表强于链表, 修改数据链表强于顺序表

**链表典型运用与套路**:  
1. 双指针(快慢指针):  
    1. 找到倒数第n个结点:  
       1. 快指针先走n步
       2. 慢指针开始走
       3. 两者同步更新
       4. 快指针指向NULL时, 慢指针走到倒数第n个结点
    2. 找到中间结点  
       1. 快指针初始指向head->next,慢指针初始指向head
       2. 快指针一次走两步,慢指针一次走一步
       3. 直到快指针或快指针所指向的结点指针域所指向的结点为空
       4. 此时慢结点所指向的结点为中间结点
    3. 判断链表中是否有环  
       1. 快指针走两步, 慢指针走一步  
       2. 两个能指向同一个结点说明有环  
        ```C
        int isCycle(Node *head){
            Node* fast = head;
            Node* slow = head;

            while(fast != NULL && fast -> next != NULL){
                fast = fast -> next -> next;
                slow = slow -> next;
                if(fast == slow){
                    return 1;
                }
            }
            return 0;
        }
        ```
        3. 此时开始计数, 记录多少步后(记作count)快慢指针再一次重合, 即为环的周长有多少个结点
        4. 然后让快慢指针同时指向头指针, 快指针比慢指针先走count步, 然后同步走, 等两个指针再次相遇时就是环的入口
        ```C
        Node* findBegin(Node* head){
            Node* fast = head;
            Node* slow = head;

            while(fast != NULL && fast -> next != NULL){
                fast = fast -> next -> next;
                slow = slow -> next;
                if(fast == slow){
                    Node* p = fast;
                    int count = 1;
                    while(p -> next != slow){
                        count++;
                        p = p -> next;
                    }
                    fast = head;
                    slow = head;

                    for(int i = 0; i < count; i++){
                        fast = fast -> next;
                    }
                    while(fast != slow){
                        fast = fast -> next;
                        slow = slow -> next;
                    }
                    return slow;
                }
            }
            return NULL;
        }
        ```
2. 反转单链表  
    1. 三个指针(first, second, third)  
    2. first初始指向空值, second初始指向首结点, third指向second指向的结点的下一个结点  
    3. second所指向的结点的指针域指向first所指向的结点
    4. first指向second指向的结点, second指向third指向的结点, third指向third所指向的结点的指针域所指向的结点(即全体往后挪)
    5. 重复以上过程, 直到second为空值
    6. 头指针指向first  
    典型应用
    ```C
    void reverseList(Node* head){
        Node* first = NULL;
        Node* second = head -> next;
        Node* third;

        while(second != NULL){
            third = second -> next;
            second -> next = first;
            first = second;
            second = third;
        }
        *head = first;
    }
    ```
**单链表的局限性: 不能回头!!**

**双向链表:**  
两个指针: 一个指向直接后继, 一个指向直接前驱  
数据域  

典型的双向链表结点实现:  
```C
typedef int ElemType;

typedef struct node{
    ElemType data;
    struct node *prev, *next;
}Node;
```

**双向链表头插法**:  
典型实现:  
```C
int insertHead(Node* L, ElemType e){
    Node *p = (Node*)malloc(sizeof(Node));
    p -> data = e;
    p -> prev = L;
    p -> next = L -> next;
    if(L -> next != NULL){
        L -> next -> prev = p;
    }
    L -> next = p;
    return 1;
}
```

**双向链表尾插法**:  
典型实现:
```C
Node * insertTail(Node *tail, ElemType e){
    Node *p = (Node*)malloc(sizeof(Node));
    p -> data = e;
    p -> prev = tail;
    tail -> next = p;
    p -> next = NULL;
    return p;
}
```

**双向链表指定位置插入数据**:  
典型实现:  
```C
int insertNode(Node *L, int pos, ElemType e){
    Node *p = L;
    int i = 0;
    while(i < pos - 1){
        p = p -> next;
        i++;
        if(p == NULL){
            return 0;
        }
    }

    Node *q = (Node*)malloc(sizeof(Node));
    q -> data = e;
    q -> prev = p;
    q -> next = p -> next;
    p -> next -> prev = q;
    p -> next = q;
    return 1;
}
```  

**双向链表指定位置删除结点**:  
典型实现:  
```C
int deleteNode(Node *L, int pos){
    Node *p = L;
    int i = 0;
    while(i < pos - 1){
        p = p -> next;
        i++;
        if(p == NULL){
            return 0;
        }
    }

    if(p -> next == NULL){
        printf("要删除的位置错误\n");
        return 0;
    }
    
    Node *q = p -> next;
    p -> next = q -> next;
    q -> next -> prev = p;
    free(q);
    return 1;
}
```
## 3. 栈和队列
### 3.1 栈的定义与实现
**栈**是只在表尾进行插入或删除操作的线性表, 表尾称为栈顶, 表头称为栈底, 不含任何元素的栈叫空栈  

栈具有**后进先出**的性质(Last In First Out, LIFO)  

#### 3.1.1 栈的顺序结构实现
**定义**  
```C
#define MAXSIZE = 100
typedef int ElemType;

typedef struct{
    ElemType data[MAXSIZE];
    int top;
}Stack;
```

**初始化**  
```C
void initStack(Stack *s){
    s -> top = -1;
}
```

**压栈**  
```C
int push(Stack *s, ElemType e){
    if(s -> top >= MAXSIZE -1){
        printf("满了\n");
        return 0;
    }
    s -> top++;
    s -> data[s -> top] = e;
    return 1;
}
```

**出栈**  
```C
ElemType pop(Stack *s, ElemType *e){
    if(s -> top == -1){
        printf("空的\n");
        return 0;
    }
    *e = s -> data[s -> top];
    s -> top--;
    return 1;
}
```

#### 3.1.2 栈的链式结构实现  
**单结点定义**  
```C
typedef int ElemType;

typedef struct stack{
    ElemType data;
    struct stack * next;
}Stack;
```

**判断栈是否为空**  
```C
int isEmpty(Stack *s){
    if(s -> next == NULL){
        printf("空的\n");
        return 1;
    }
    else{
        return 0;
    }
}
```

**出栈**  
```C
int pop(Stack *s, ElemType *e){
    if(s -> next == NULL){
        printf("空的\n");
        return 0;
    }
    *e = s -> next -> data;
    Stack *q = s -> next;
    s -> next = q -> next;
    free(q);
    return 1;
}
```
**入栈**  
链式实现的栈的入栈与链表的头插法相同  

### 3.2 队列定义与实现
**队列**是一种**先进先出**(First In First Out, FIFO)的线性表, 只允许在表的一端进行插入, 另一端进行删除

允许插入的一端叫**队尾**, 允许删除的一端叫**队头**  

#### 3.2.1 队列的顺序结构实现  
**结构定义**  
典型应用  
```C
#define MAXSIZE 100
typedef int ElemType;
typedef struct{
    ElemType data[MAXSIZE];
    int front;
    int rear;
}Queue;
```

**初始化**  
典型应用  
```C
void initQueue(Queue * Q){
    Q -> front = 0;
    Q -> rear = 0;
}
```

**判断队列是否为空**  
典型应用  
```C
int isEmpty(Queue *Q){
    if(Q -> front == Q -> rear){
        printf("空的\n");
        return 1;
    }
    else{
        return 0;
    }
}
```

**出队**  
典型应用  
```C
ElemType dequeue(Queue *Q){
    if(Q -> front == Q -> rear){
        printf("空的\n");
        return 0;
    }
    ElemType e = Q -> data[Q -> front];
    Q -> front++;
    return e;
}
```

**入队**  
典型应用  
```C
int equeue(Queue *Q, ElemType e){
    if(Q -> rear >= MAXSIZE){
        if(!queueFull(Q)){
            return 0;
        }
    }
    Q -> data[Q -> rear] = e;
    Q -> rear++;
    return 1;
}

//即使rear指向最后队列也可能还没满, 这个函数用于挪动队列
int queueFull(Queue *Q){
    if(Q -> front > 0){
        int step = Q -> front;
        for(int i = Q -> front; i <= Q -> rear; i++){
            Q -> data[i - step] = Q -> data[i];
        }
        Q -> front = 0;
        Q -> rear = Q -> rear - step;
        return 1;
    }
    else{
        printf("真的满了\n");
        return 0;
    }
}
```

不难看出想要充分利用顺序队列中的空间就不得不对所有元素进行挪动, 这无疑是一种低效的方法, 因此, 循环队列的重要性得以凸显

#### 3.3.1 如何计算后缀表达式  
从左往右扫描输入的字符串, 如果是数字则压入栈中, 如果是操作符则出栈两个数字, 先出栈的数字放在该运算符的右边, 后出栈的放在该操作符的左边, 计算运算的结果并将结果压入栈中, 循环上述过程, 遇到'\0'输出栈中的结果

#### 3.3.2 中缀表达式如何转换为后缀表达式
1. 初始有一个栈, 在栈里有一个优先级为0的元素  
2. 接下来扫描中缀表达式
3. 如果是数字则直接输出
4. 如果是运算符, 判断优先级, 如果优先级大于栈顶元素, 压入栈中, 否则将栈顶元素出栈输出, 然后将当前运算符压入栈中  
5. 如果是左括号, 左括号在栈外的优先级最高, 在栈中的优先级为0  
6. 如果是右括号, 则持续出栈并输出, 直到栈顶为左括号, 然后将左括号也出栈(不输出)
7. 遇到'\0', 输出所有栈中的内容

## 4. 树与二叉树
### 4.1 树与二叉树的定义
#### 4.1.1 树
树是一个或多个结点的有限集合  

存在一个称为根的特定结点, 其余结点被分为n个互不相交的集合, 其中每一个集合都是一棵树, 称为根节点的子树  

**结点**: 树中的一个独立单元  
**结点的度**:结点拥有的子树数被称为结点的度  
**树的度**: 树内各结点度的最大值  
**叶子**: 度为0的结点或终端结点  
**双亲和孩子**: 结点的子树的根称为该结点的孩子, 相应地, 该结点称为孩子的双亲  
**非终端结点**: 度不为0的结点  
**层次**: 结点的层次从根开始定义, 根为第一层, 根的孩子为第二层, 依次类推  

**树的基本性质**:  
1. 树中所有结点数等于所有结点的度数之和加一
2. 对于度为m的树, 第i层上最多有$m^{i-1}$个结点
3. 对于高度为h, 度为m的树, 最多有($m^h-1$)/($m-1$)个结点

#### 4.1.2 二叉树  
**二叉树**:  
每个结点的度最多为2, 有且只有一个根结点, 二叉树的子树有左右之分, 其次序不能任意颠倒

**二叉树的性质**:  
1. 二叉树的第i层最多有$2^{i-1}$个结点
2. 深度为k的二叉树最多有$2^k-1$个结点
3. 对于任何非空二叉树, 如果叶子结点的个数为$n_0$, 而度为2的结点数为$n_2$, 则$n_0=n_2+1$

**特殊二叉树**:  
1. 满二叉树: 深度为k且含有$2^k-1$个结点的二叉树
   1. 所有叶子结点只能出现在最后一层
   2. 对于同样深度的二叉树, 满二叉树的结点个数最多, 叶子结点的个数也是最多的
   3. 如果对满二叉树进行编号, 根结点从1开始, 从上到下, 从左到右, 对于编号为i的结点, 若存在左孩子, 则左孩子的编号为2i, 右孩子为2i+1
2. 完全二叉树:深度为k的, 有n个结点的二叉树, 当且仅当每一个结点都与深度为k的满二叉树中编号从1至n的结点一一对应时, 称之为完全二叉树
   1. 叶子结点只可能在层次最大的两层上出现
   2. 对任一结点, 若其右分支下的子孙的最大层次为l, 则其左分支下的子孙的最大层次必为l或l+1
   3. **没有左子树, 不能有右子树, 上一层没有铺满, 不能有下一层**
   4. 完全二叉树的性质
      1. 具有n个结点的完全二叉树的深度为$\log_2 n+1$(向下取整)
      2. 如果对一颗有n个结点的完全二叉树(其深度为$\log_2 n+1$(向下取整))的结点按层序编号(从第一层到第$\log_2 n+1$(向下取整层), 每层左到右), 则对任一结点i($1\le i \le n$), 以下结论成立 
         1. 如果i = 1, 则结点i是二叉树的根, 无双亲; 如果结点i>1则其双亲是结点i/2(向下取整)
         2. 如果2i>n, 则结点i无左孩子(结点i为叶子结点); 否则其左孩子是结点2i
         3. 如果2i + 1 > n, 则结点i无右孩子; 否则其右孩子的结点是2i + 1 

### 4.2 二叉树的链式存储
除了满二叉树和完全二叉树以外, 使用顺序结构存储都比较浪费空间, 因此首选链式存储

#### 4.2.3 线索二叉树  
通过遍历可以得到二叉树的线性排列, 但这样的线性序列只有在遍历时才能得到, 将二叉树线索化得到线索二叉树可以解决这个问题  
**线索化**:利用叶节点的空余空间记录前驱和后继

线索二叉树的目的是为了能像遍历双向链表一样轻松, 而不需要每次都使用递归

要先确定顺序才能加线索  

**线索二叉树的使用场景**: (老旧)操作系统的文件管理系统返回上一层

右孩子为空值的结点用来记录后继  
左孩子为空值的结点用来记录前驱

**结论**: 如果一颗二叉树有n个结点, 那么就一定有n+1个空余空间

**线索二叉树结点典型实现**  
```C
typedef char ElemType;

typedef struct ThreadNode{
    ElemType data;
    struct ThreadNode *lchild;
    struct ThreadNode *rchile;
    int ltag;//ltag为0时, 指向该结点左孩子, 为1时, 指向该结点的前驱
    int rtag;//rtag为0时, 指向该结点右孩子, 为1时, 指向该结点的后继
}ThreadNode;

typedef ThreadNode* ThreadTree;
```

**中序遍历线索化**  
1. 头结点的lchild指向二叉树的根  
2. 头结点的rchild指向遍历的最后一个结点  
3. 第一个结点的lchild指向头结点  
4. 最后一个结点的rchild指向头结点  

**典型实现**  
```C
#include <stdio.h>
#include <stdlib.h>

typedef char ElemType;

typedef struct ThreadNode{
    ElemType data;
    struct ThreadNode *lchild;
    struct ThreadNode *rchild;
    int ltag;
    int rtag;
}ThreadNode;

typedef ThreadNode* ThreadTree;

char str[] = "ABDH##I##EJ###CF##G##";//前序字符串
int idx = 0;

ThreadTree prev;

//创建树
void createTree(ThreadTree *T){
    ElemType ch;
    ch = str[idx++];
    if(ch == '#'){
        *T = NULL;
    }
    else{
        *T = (ThreadTree)malloc(sizeof(ThreadNode));

        (*T) -> data = ch;

        createTree(&(*T) -> lchild);
        if((*T) -> lchild != NULL){
            (*T) -> ltag = 0;
        }

        createTree(&(*T) -> rchild);
        if((*T) -> rchild != NULL){
            (*T) -> rtag = 0;
        }
    }
}

//具体线索化
void threading(ThreadTree T){
    if(T != NULL){
        threading(T -> child);
        if(T -> lchild == NULL){
            T -> ltag = 1;
            T -> lchild = prev;
        }
        if(prev -> rchild == NULL){
            prev -> rtag = 1;
            prev -> rchild = T;
        }
        prev = T;
        threading(T -> rchild);
    }
}

//开始线索化
void inOrderThreading(ThreadTree *head, ThreadTree T){
    //头结点
    *head = (ThreadTree)malloc(sizeof(ThreadNode));
    (*head) -> ltag = 0;
    (*head) -> rtag = 1;
    (*head) -> rchild = (*head);

    if(T == NULL){
        (*head) -> lchild = *head;
    }
    else{
        (*head) -> lchild = T;
        prev = (*head);

        threading(T);

        //最后一个结点线索化
        prev -> rchild = *head;
        prev -> rtag = 1;

        //头结点右孩子指向最后一个结点
        (*head) -> rchild = prev;
    }
}

//使用线索进行中序遍历
void inOrder(ThreadTree T){
    ThreadTree curr;
    curr = T -> lchild;

    while(curr != T){
        while(curr -> ltag == 0){
            curr = curr -> lchild;
        }
        printf("%c ", curr -> data);

        while(curr -> rtag == 1 && curr -> rchild != T){
            curr = curr -> rchild;
            printf("%c ", curr -> data);
        }
        curr = curr -> rchild;
    }
    printf("\n");
}

int main(){
    ThreadTree head;
    ThreadTree T;
    //创建
    createTree(&T);
    //线索化
    inOrderThreading(&head, T);
    //基于线索遍历
    inOrder(head);

    return 0;
}
```

### 4.3 哈夫曼树
**哈夫曼树基本概念**:  
1. **路径**: 从树上一个结点到另一个结点之间的分支  
2. **路径长度**: 路径上的分支数目  
3. **树的路径长度**: 从树根到每一个结点的路径长度之和  
4. **结点的权**: 在实际应用中, 给树中的结点赋予表示某种含义的数值  
5. **结点的带权路径长度**: 从该结点到树根之间的路径长度与该结点权的乘积
6. **树的带权路径长度(WPL)**: 树中所有叶结点的带权路径长度之和

带权路径长度最小的二叉树被称为二叉树  

**构造哈夫曼树**  
1. 先把有权值的叶子结点按从小到大的顺序排列成一个有序序列  
2. 取两个最小权值的结点作为一个新结点$N_1$的子结点
3. 将结点$N_1$替换刚刚取出的两个结点, 并加入到有序序列中  
4. 重复第二步

**哈夫曼编码**  
为了解决当年远距离通信的数据传输最优化问题

把字母按照使用频率赋权, 构造一棵哈夫曼树, 然后左路径记为0, 右路径记为1, 按照从根到对应字母的路径进行编码(效率最高的编码)

哈夫曼树一定没有度为1的结点

### 4.4 树与二叉树的转换
**树转换成二叉树**  
1. 加线, 在所有兄弟结点之间加一条线   
2. 去线, 对树中的每一个结点, 只保留它与第一个孩子结点之间的连线, 删除它与其它孩子结点之间的连线
3. 层次调整, 以树的根结点为轴心, 将整棵树顺时针旋转一定角度, 使其层次分明, 注意第一个孩子是二叉树结点的左孩子, 兄弟转过来的孩子是结点的右孩子

**二叉树转换成树**  
1. 加线, 若某个结点的左孩子存在, 则将这个左孩子的所有右孩子结点都作为此结点的孩子, 将该结点与这些右孩子结点用线连起来
2. 去线, 删除二叉树中所有结点与其右孩子结点的连线
3. 转一下

### 4.5 森林与二叉树的转换
**森林转换成二叉树**  
1. 将森林中的每一棵树都先转换为二叉树
2. 第一棵二叉树不动, 从第二棵树开始, 依次把后一棵二叉树的根结点作为前一棵二叉树的根结点的右孩子, 然后用线连起来

**二叉树转换为森林**  
1. 从根结点开始, 若右孩子存在, 则把右孩子结点的连线删除, 然后把互不相连的拆开  
2. 接下来把每一个独立的二叉树转换为树

**树的遍历**  
1. 先根遍历(类似前序遍历)  
2. 后根遍历(类似后序遍历)

**一棵树的后根遍历结果与这棵树所转化成的二叉树的中序遍历结果相同**

**一棵树的前根遍历结果与这棵树所转化成的二叉树的前序遍历结果相同**

树, 森林, 二叉树遍历结果关系如下图所示
| 树       | 森林     | 二叉树   |
| -------- | -------- | -------- |
| 先根遍历 | 前序遍历 | 前序遍历 |
| 后根遍历 | 中序遍历 | 中序遍历 |

**一棵树的结点数都比边数大一**  

**计算二叉树的最大深度(也可用于层序遍历, 则一层一层的遍历)**  
典型实现  
```C
int maxDepth(TreeNode* root){
    if(root == NULL){
        return 0;
    }

    int depth = 0;
    Queue *q = initQueue();
    equeue(q, root);

    while(!isEmpty(q)){
        int count = queueSize(q);//记录队列中的元素个数
        while(count > 0){
            TreeNode* curr;
            dequeue(q, &curr);
            if(curr -> lchild != NULL){
                equeue(q, curr -> lchild);
            }
            if(curr -> rchild != NULL){
                equeue(q, curr -> rchild);
            }
            count--;
        }
        depth++;
    }
    return depth;
}

//获取队列的元素数量
int queueSize(Queue *Q){
    if(!isEmpty(Q)){
        return Q -> rear - Q -> front;
    }
    else{
        return 0;
    }
}
```

## 5. 图
**图最多解决的是路径搜索问题**  

### 5.1 图的基本概念  
**图的定义**： 图是有顶点的有穷非空集合和顶点之间边的集合组成的。通计为G(V, E)，其中G表示一个图，V表示图G中顶点的集合，E表示边的集合，V(G)和E(G)通常表示为G的顶点集合和边集合，允许没有边，不允许没有顶点

图分为有向图和无向图，边有方向的叫有向图，否则叫无向图，有向图的边一般叫弧，带箭头的叫弧头，不带的叫糊尾  

**简单图与多重图**：  
1. 限制一：图中不能有从顶点到自身的边
2. 限制二： 同一条边在图中不能出现两次或两次以上  
不满足以上两条限制的图称为多重图，满足就叫简单图  

**以下所有图均为简单图**  

**完全图**：具有最多边数的图  
对于一个具有n个顶点的无向完全图，边数量的最大值为n(n - 1)/2  
对于一个具有n个顶点的有向完全图，边数量的最大值为n(n - 1)  

**路径和路径长度**  
路径：从一个顶点开始，经过一系列的边到达另一个顶点形成的顶点序列  
路径长度：路径上边的条数  
回路（环）：起点和终点相同的路径  

**简单路径和简单回路**  
简单路径：如果路径中不出现相同的顶点，称为简单路径  
简单回路：除第一个顶点和最后一个顶点外，其余顶点不重复出现的回路称为简单回路  

**顶点的度**  
度：对于无向图，顶点的度指的是与顶点相关联边的数目  
入度：在有向图中，对于顶点v，箭头指向v的边的数目  
出度：在有向图中，对于顶点v，从该顶点出发的边的数目  

**度与边的关系**  
在无向图中，假设具有n个顶点，e条边，图中所有顶点度数之和等于边数的两倍
对于有向图，所有顶点的出度之和与入度之和相等，弧的数量也相等

**子图**  
即为原图的子集，顶点是原图的子集，边也是原图的子集

**连通图**  
在无向图中，如果从顶点V到顶点W有路径，则称顶点V到顶点W是连通的。如果对于图中任意两个顶点都是连通的，则此图为连通图  

**联通分量**  
无向图中的极大连通子图称为连通分量  
连通分量为子图  
子图为连通图  
连通子图含有极大顶点数  
具有极大顶点的连通子图包含依附于这些顶点的所有边  
**如果一个连通子图添加任意一个顶点就不是连通子图，那么这个连通子图就是极大连通子图（类比极大无关组）**

**强连通图**  
在有向图中，对于每一对顶点v和w，从v到w和从w到v都有路径，则称该有向图是强连通图。  

有向图中的极大强连通子图称为有向图的强连通分量  

**生成树**  
生成树：指含有图中全部顶点的极小连通子树
**注意：包含所有顶点n，但只有足以构成一棵树的n - 1条边**


**边的权和网**  
在一个图中，每条边可以标注上一个代表某种含义的数值，该数值称为这个边的权值  
网：边上带权值的图，也称为带权图  

### 5.2 图的存储结构  
**邻接矩阵（无向）**  
一个二维数组，表示两个顶点之间是否有连线，有用1表示，无用0表示对于无向图，这是一个对称矩阵  

**邻接矩阵（有向）**  
纵轴表示弧尾，横轴表示弧头

**邻接矩阵（带权值）**  
没有关联的用无穷表示，其他用权值表示  

**邻接表（无向）**  
通过链表将与自己相连的结点连接在一起

**邻接表（有向）**  
通过链表按出度方向表示所有与自己相连的结点  

**逆邻接表**  
按入度方向表示所有与自己相连的结点

**但不管是邻接表还是逆邻接表对于有向表来说都无法既涵盖入度方向的相连结点又涵盖出度方向的相连结点，所以我们引入了十字链表**  

**十字链表**  
顶点结构：data, firstin(入边表头指针), firstout(出边表头指针)  
边结构：tailvex(弧尾), headvex(弧头), headlink(头指针), taillink(尾指针)  
**横向表示出边，纵向表示入边**

**邻接多重表(针对无向图)**  
ivex和jvex是某一条边连接到两个顶点的下标  
ilink指同连接顶点jvex的下一条边  
jlink指同连接顶点jvex的下一条边

### 5.4 最小生成树  
#### 5.4.1 普里姆(Prim)算法(贪心)(局部最优解)(关注顶点) 
每一步都选权值最小的  
1. 找一个顶点
2. 观察与其相连的路径(加入候选路径)
3. 从候选路径中选择权值最小的那条(把与该路径相连的顶点加入已访问组中)(不能选择已经访问的顶点)
4. 选完之后把与新结点相连的路径加入候选路径
5. 跳回第二步

```C
void prim(Mat_Grph* G){
    int i, j, k;
    int min;
    int weight[MAXSIZE];//侯选边
    int vex_index[MAXSIZE];//值表示出发点，下标表示到达点

    //先从顶点A开始
    weight[0] = 0;//vex_index某点与下标对应点的权重
    vex_index[0] = 0;

    for(i = 1; i < G -> vertex_num; i++){
        weight[i] = G -> arc[0][i];
        vex_index[i] = 0;
    }

    for(int i = 1; i < G -> vertex_num; i++){
        min = MAX;
        j = 0;
        k = 0;
        //找到最小的，与之相连
        while(j < G -> vertex_num){
            if(weight[j] != 0 && weight[j] < min){
                min = weight[j];
                k = j;
            }
            j++;
        }
    

        printf("(%c, %c)\n", G -> vertex[vertex_index[k]], G -> vertex[k]);
        weight[k] = 0;

        //找新连接点结点的待选路径
        for(j = 0; j < G -> vertex_num; j++){
        //与新结点可连接的边进行比较，有比weight小的就替换
            if(weight[j] != 0 && G -> arc[k][j] < weight[j]){
                weight[j] = G -> arc[k][j];
                vex_index[j] = k;
            }
        }
    }
}

    prim(&G);
    return 0;
}
```

```C
void swap(Edge* edges, int i, int j){
    int temp;
    temp = edges[i].begin;
    edges[i].begin = edges[j].begin;
    edges[j].begin = temp;

    temp = edges[i].end;
    edges[i].end = edges[j].end;
    edges[j].end = temp;

    temp = edges[i].weight;
    edges[i].weight = edges[j].weight;
    edges[j].weight = temp;
}

void sortEdges(Edge edges[], int edge_num){
    for(int i = 0; i < edge_num; i++){
        for(int j = i + 1; j < edge_num; j++){
            if(edges[i].weight > edges[j].weight){
                swap(edges, i, j);
            }
        }
    }
}

int find(int* parent, int index){
    while(parent[index] > 0){
        index = parent[index];
    }
    return index;
}

void Kruskal(Mat_Grph G){
    Edge edges[MAXEDGE];
    int k = 0;

    for(int i = 0; i < G.vertex_num; i++){
        for(int j = i + 1; j < G.vertex_num; j++){
            if(G.arc[i][j] < MAX){
                edges[k].begin = i;
                edges[k].end = j;
                edges[k].weight = G.arc[i][j];
                k++;
            }
        }
    }

    sortEdges(edges, G.edge_num);
```

### 5.5 最短路径  
单源最短路径：从某一个顶点出发，到图中所有其他顶点的最短路径

#### 5.5.1 迪杰斯特拉(Dijkstra)算法  
**核心思想**：逐层扩展
1. 从起点出发初始化到各顶点的距离(起点为0，其他为无穷大)
2. 每次从未访问的顶点中，找出当前距离起点最近的那个
3. 以这个顶点为中转，尝试放松它所有相邻顶点的距离
4. 重复第2、3步，直到所有顶点都被确定最短距离为止

**注意：该算法不是以终点为导向的，而是以起点为核心，全图向外拓展的策略。算法可以获取从起点到全图任意一个点的最短路径，当指定一个终点时，算法会直接告诉你从起点到终点的最短路径**

#### 5.2.3 拓扑排序
在一个表示工程的有向图中，用顶点表示活动，用顶点表示活动，用弧表示活动之间的优先级，这样的有向图为顶点表示活动的网，我们称为AOV网(Activity On Vertex Network)

**拓扑排序是解决怎么遍历一张有向图的问题**

思路：持续寻找入度为0的顶点，然后入栈，出栈

```C
    if(top > MAXSIZE){
```
